# Smart Path Finder — Implementation Plan (v3 — Final)

## Overview

Build a Python-based Smart Path Finder that computes shortest paths (by distance and travel time) on a realistic road network graph. All data (nodes, edges, coordinates) are generated by us — no external data sources. Persisted in CSV so map stays stable across runs.

---

## Key Decisions (Finalized)

| Decision | Choice |
|---|---|
| Coordinates | Generated by us — realistic scatter, NOT a grid, NOT from OSM |
| Node names | Meaningful names from a pool of real-world-style street/place names |
| Edges | Algorithmically generated — k-NN + regional + highway links |
| Distance | Haversine from generated lat/lon coordinates (meters) |
| Original Dijkstra | O(V²) array scan **with early termination** |
| Optimized Dijkstra | Binary heap + lazy deletion + early termination |
| Departure hour | User-specified, **default 8:00 AM** |
| Evaluation | Per-query metrics: runtime, explored nodes, relaxed edges, heap ops |
| Libraries | None — all from scratch (custom heap, CSV parsing, etc.) |

## Implementation Details

### 1. Node Generation (`map_generator.py`)

- **~2,000 nodes** placed in a virtual city region (~10.7–10.85 lat, ~106.6–106.75 lon, roughly HCMC-scale)
- **Scatter placement** using clustered random distribution:
  - Define ~15–20 "neighborhoods" as cluster centers
  - Each cluster gets 80–150 nodes scattered around its center (Gaussian spread)
  - This creates dense urban cores and sparser suburban areas — NOT a grid
- **Minimum spacing**: enforce ≥ 50m between any two nodes
- **Meaningful names**: combine from pools like:
  - Street names: "Nguyen Hue", "Le Loi", "Tran Hung Dao", "Pasteur", "Hai Ba Trung", etc.
  - Place types: "Junction", "Square", "Market", "Station", "Park", "Bridge"
  - Pattern: `"<Street1> / <Street2>"` or `"<Place> <Type>"`

### 2. Edge Generation (`map_generator.py`)

- **K-nearest-neighbor** (k=3–5) for backbone connectivity
- **Regional links** within 2–5 km with decreasing probability
- **Highway links** (~5%) connecting distant nodes (5–15 km)
- **~70% bidirectional**, ~30% one-way
- **Distance** = Haversine(nodeA, nodeB) in meters
- **Travel times**: 24 values per edge based on road type + congestion profile
- **Graph connectivity** verified (all reachable from any node)

### 3. Heap (`heap.py`) — Custom binary min-heap from scratch

### 4. Dijkstra (`dijkstra.py`) — Original O(V²) + Optimized O((V+E)log V), both with early termination

### 5. CSV I/O (`csv_io.py`) — Load/save, startup checks existing files

### 6. Query (`query.py`) — Source, dest, departure_hour (default=8), avoid lists

### 7. Main (`main.py`) — Interactive CLI with name search, both algorithms, display metrics
